# STM32 Keil Workflow Skill

## 简介
这是一个专门为 STM32 平台和 Keil MDK-ARM 工具链打造的自动化固件开发 Skill。支持通过 STM32 SPL（标准外设库）和 HAL 库创建工程、分析 CubeMX `.ioc` 文件、编写代码并自动调用 Keil 进行编译、烧录和 Debug。

## 功能内容
- **智能工程创建**：支持从零生成模板项目，或导入 STM32CubeMX `.ioc` 文件自动生成 Keil (`.uvprojx`) 工程结构。
- **模板与驱动生成**：内置常见外设（USART, SPI, I2C 等）的标准驱动模板代码，快速生成可用外设模块。
- **编译与自动化修复**：支持在后台调用 Keil 的 UV4.exe 进行编译，自动解析构建日志，遇到报错可自动根据特征库尝试修复。
- **CMSIS-DAP / ST-Link 烧录支持**：编译成功后可直接将固件烧录进目标单片机。
- **HardFault 诊断**：集成高级调试工作流，可追踪 HardFault 发生的源代码行数并提供修复建议。

## 如何使用
将本文件夹引入 Claude 的 Skill 环境即可。你可以通过自然语言下达指令：
- "创建一个基于 STM32F407 的串口通信项目，使用 HAL 库。"
- "帮我解析这个 `.ioc` 文件并生成 Keil 工程代码。"
- "编译当前 Keil 项目并烧录，如果有报错请自动修复。"

## 注意事项
1. **工具链要求**：必须在本地或目标机器安装好 **Keil MDK-ARM (UV4)**，且芯片对应的 Pack 已经安装完毕。
2. **配置确认**：涉及真实硬件管脚分配时，Agent 在自动生成前可能会要求你进行最终确认，请仔细检查以防硬件短路或损坏。
3. **安全恢复策略**：发生编译错误时，Skill 会自动分析并请求修复策略，部分破坏性修改需要用户确认才能应用。
