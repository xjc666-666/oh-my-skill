# MSPM0G3519 Development Skill

## 简介
专门针对德州仪器（TI）MSPM0G3519（Cortex-M0+ 架构）的固件开发自动化工作流。本 Skill 整合了 Keil MDK-ARM、TI SysConfig 图形化配置工具以及 DriverLib，实现代码生成、错误检查、编译、CMSIS-DAP 烧录的全链路自动化。

## 功能内容
- **SysConfig 深度集成**：能根据自然语言需求自动修改 `.syscfg` 文件，增删外设、配置引脚及时钟树，并在后台自动生成初始化代码。
- **项目骨架与驱动**：包含 MSPM0G3519 的标准工程结构及常见外设（如 OLED、矩阵键盘、UART、ADC、PWM）的驱动库。
- **三级代码质量检查**：强制对代码进行语法、逻辑和平台规范的多维度检查，杜绝因疏忽导致的低级错误。
- **ADC/DAC 与模拟校准支持**：针对依赖模拟基准电压（VREF）的外设做了特殊约束与优化。

## 如何使用
在 Claude 交互环境中使用，可以直接输入类似如下的需求：
- "为 MSPM0G3519 生成一个带 OLED 和矩阵键盘中断的工程。"
- "配置 SysConfig，开启一个 DAC12，并编写相应的输出代码。"
- "编译并使用 CMSIS-DAP 烧录当前的 MSPM0G3519 工程。"

## 注意事项
1. **硬件引脚铁律**：由于开发板的特定接线（如 OLED 和 Keyboard），这部分引脚是固定的。对于其他自定义外设（UART、ADC 等），Claude 会严格向你确认引脚分配，请务必配合确认。
2. **VREF 实测值**：使用 ADC/DAC 等模拟外设时，**必须使用万用表实测开发板上的 VREF 电压**（因个体差异），并将实际值输入 SysConfig 否则计算可能出现严重偏差。
3. **DAC与放大器**：通过本 Skill 配置 DAC 时，会自动帮你开启输出缓冲放大器，以确保引脚能够测得实际电压。
